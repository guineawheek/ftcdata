Welcome to FTCData Pre-Alpha Edition!

(Yet Another FTC Event Results Website)

Currently serving {{team_count}} teams, {{event_count}} events, {{match_count}} matches, and {{award_count}} awards from Hot Shot! through Rover Ruckus, aiming to compile as comprehensive of an archive of the FIRST Tech Challenge's competition history as one can possibly manage in one place, for the preservation of this program's heritage.

As this website is in pre-alpha, parts of this website may be broken, completely nonfunctional, or change on a moment's whim.

Frequently Mentioned Things:

Why though? Aren't there already websites like this?

The current state of FTC match and event data is hilariously fragmented across many different sources of varying quality and likelihood of staying up, from blog posts on an affiliate partner's website that might get wiped clean in 2 years, to shiny, organized affairs such as FTCPenn's or the Missouri FTC Scores website, or, if one is lucky, uploaded to either The Orange Alliance through FIRST or FTCScores through their proprietary (eww!) live uploader. (However, neither cover data pre-dating Velocity Vortex.)

Out of these, The Orange Alliance is certainly one of the stronger solutions. However, due to their contract with FIRST in order to be able to provide data gathered from affiliate partners who actually bother submitting data to them (like they should!) in the first place, they cannot display match/awards data from other sources. Plus, I personally believe their UI flows leave something to be desired...and competition is good.

While FTCScores certainly sports a prettier UI, it's entire stack is proprietary software, closing the potential community of contributors to those currently working on their project. Plus, if they die at any time, there will be nobody around to revive them.

Given that it's main purpose is primarily serving data live straight from events anyway, rather than being a general purpose match results website like TOA, archival of past events is not really a priority for them.

For obvious reasons, regional affiliate partners typically only display data from their own regions on their websites. Sometimes, the cheekier ones think this is a good substitute for submitting their events to FIRST. (it's not, you often lose a lot of detail.) That said, FIRST historically didn't even take meet data, so there might be something to be said for that one.

Archiving event histories is pretty important. One's own team history is something one might want to know about. One may want to know, for example, if one's was Very Good 5 years ago, but now seems to struggle to make it to states. Being downstream of all these sources on our own platform allows us to not only present and preserve all their data without having to deal with the political reasons for why any one of them might not be able to archive FTC's history. Our aim for this project is to be the spiritual successor to FTCRoot and cheer4ftc's ftc-data repository - with the aim of recording every single bit of match and awards data we can get our hands on.

Plus, we have an opportunity to build a faster, more intuitive UI building upon an already established platform...speaking of which....

This looks exactly like a certain Alliance Of Blue Color's website.

That's because many of TBA's HTML templates were used (and heavily modified) in this project to build the UI you see right now. TBA's code and HTML templates are open source under the MIT License just like FTCData, and since it's UI is already pretty good (compared to what I might write from scratch), I figured I might as well use them. I hope you agree on my assessment of TBA's UI, but understand if you don't.

The backend, like TBA, is written in Python, however, most of these pages you see are likely pregenerated static files that are served through nginx. Structurally, while some general layout and design cues were borrowed from TBA (and a few select helper files) the codebase is otherwise very different.

If you see any references to TBA or FRC on this website, that's a bug. If you are reading this, I should've attempted to strip out all of them already. Please report it if you do see them.

Is there an API?

As this software is very much in a pre-alpha state, much of the main functionality of the website isn't even finished, much less an API. However, I can offer the next best thing: regular Postgres backups of the entire data set in pg_dump format.
